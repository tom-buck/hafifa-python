# Logging

## Understanding the problem and basic logging

בין הדברים הכי בסיסים שלומדים שפת תכנות הוא להדפיס :

```bash
print("hello world")
```

בדרך כלל מתפתח הרגל להשתמש בהדפסות לכול דבר שנרצה - דיבוג , תיעוד וכו.

ישנם מספר בעיות עם זה : 
המידע מופיע בקונסול ולא נשמר בשום מקום.
בשביל פורמט מסוים של מידע נצטרך להשקיע זמן בשביל ליצור משהו כזה - למשל נרצה לשמור את הזמן של ההדפסה או מידע נוסף אחר.

במערכות גדולות ומרכבות יש חשוב להיות במעקב על מה קורה במערכת לכן נרצה תיעוד שנשמר בנותן לנו כמה שיותר מידע שימושי

לוגים באים לפתור את הבעיה הזאת ולהקל עלינו כמה שאפשר : 
יהיה ניתן בקלות לייצא את המידע לקובץ
ניתן בקלות להציג את זמן , הקובץ , הפונקציה והשורת קוד שממנה מוצג הלוג - מה שיעזור לנו בדיבוג
ועוד דברים שעליהם תחקרו 

עכשיו תלמדו איך משתמשים בלוגים:
[Real Python - Python Logging](https://realpython.com/python-logging/)

חשוב שתכירו לגבי רמות של לוגים (Debug, Info, Warning, Error, Critical) וקינפוג ושימוש שלהם בפרוייקט
