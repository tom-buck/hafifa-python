# Conventions

## What and Why

קונבנציות או מוסכמות כתיבה , היא הבסיס לתרבות בתכנות ששמה דגש על קריאות הקוד על פני התחכום הלוגי.
המוסמכמות האלו באות ליצור שפה ויזואלית אחידה שמאפשרת למפתחים שונים להבין קוד של אחרים.
מדובר כללים בסיסיים שמקלים עלינו בהבנה (לא מדובר בעיצוב קוד, אלא בכללים יבשים שיוצרים אחידות בדברים הכי פשוטים)
קוד נקרא הרבה יותר פעמיים מאשר שהוא נכתב.
 הקפדה על הכללים האלו מפחיתות עומס קוגנטיבי, מונעות טעויות אנוש, מקלה על תחזוקה שקורת על ידי מספר אנשים.
 (וגם עוזרת לאנשים עם OCD לא להשתגע)

## Basic Conventions


### Names


#### שמות משתנים

בפורמט `snake_case` - אותיות קטנות עם קו תחתון

```Python
    user_id = 10 
    total_amount = 60.5
```


#### שמות מחלקות

בפורמט `PascalCase` - אות גדולה בתחילת מילה, ללא קווים תחתונים

```python
    class Customer:
        pass

    class NotifcationSchuduler:
        pass
```


#### קבועים

בפורמט `UPPER_CASE` - אותיות גדולות עם קו תחתון

```python
    API_KEY = 123456
    MAX_RETRIES = 5
    FAIL_MESSAGE = "big fail"
```


#### שמות אינדקטיבים

בחרו שמות שמתארים את תוכן המשתנה , המנע משימוש של אותיות בודדות, אלא אם מודבר בלופ קצר מאוד.
שימו לב שהשם פשוט ולא ארוך בלי סיבה.
עקרון נוסף הוא בלי קיצורים ושימוש בשמות מלאים
(.כמובן שיש מקרים פרטיים לכול דבר , זכרו שהמטרה היא קריאות)

```python
#bad
    for u in users:
#good
    for user in users:

#bad
amount_of_apples = 6
#good
apples = 6

#bad
calc
#good
calculator
```


### Layout


#### הזחה (Indentation)

הזחה קוראת באמצעות `Tab`.
לא באף צורה אחרת...

```python
if number > 5:
    print(number)
^
|
tab
```


#### שורות ריקות

* **שתי שורות ריקות** לפני הגדרת פונקציה גלובלית או מחלקה.
* **שורה ריקה אחת** בין מתודות (פונקציות) בתוך מחלקה.


#### רווחים (Whitespace)

רווח אחד לפני ואחרי אופרטורים של השמה (`=`) או חישוב (`+`, `-`, `*`, `/`). אין להוסיף רווחים בתוך סוגריים.

```python
#bad
number = ( number+5 )*2
#good
number = (number + 5) * 2 
```


#### אורך שורה

מומלץ להגביל שורות ל-**79 תווים**. אם השורה ארוכה מדי, יש לשבור אותה בעזרת סוגריים או להעביר חלק מהארגומנטים לשורה חדשה.


#### ייבוא מודולים (Imports)

ייבוא תמיד יופיע בראש הקובץ, מחולק ל-3 קבוצות עם שורה ריקה ביניהן:
1.  ספריות סטנדרטיות (`os`, `sys`, `json` למשל).
2.  ספריות חיצוניות (`requests`, `pandas` למשל).
3.  ייבוא פנימי מהפרויקט.

```python
import os
import sys

import requests

from student import Student
```

### תיעוד (Docstrings)


בכדאי שהקוד יהיה ברור אנחנו עושים דבר שנקרא תיעוד - מתעדים הסברים על הקוד שלנו.
ישנו סטנדרט להסברים הללו, 

תיעוד פונקציה קורה מתחת להגדרה שלה, נפתח ב """ ונסגר ב"""
והתיעוד צריך להכיל :

*הסבר* - הסבר קצר על הפונקציה , הסבר פרטי שלא מסתכבך . 
טיפ - להתחיל במילות של פעולת ציווי כמו מחזיר , מחשב , יוצר, מבצע

*ארגומנטים* - פירוט על הארגומנטים שפונקציה מקבלת
שימו לב שה-type של הארגומנטים לא מפורט פעם נופסת , הוא אמור להיות מפורט כtype hint

*מה היא מחזירה* - 
מתאר מה היא מחזירה, אותו דבר לגבי הtype.

*איזה שגיאות היא מעלה* - 
execption שהפונציה מעלה

```python
import math

def calculate_area(radius : float) -> float:
    """
    Calculate the area of a circle based of a given radius 

    Args:
        radius : The distance from the center to the edge of the circle. 
            Must be a non negetive

    Returns:
        The total area of the circle (π * r²)

    Raises:
        ValueError: if the provided radius is a negtive number
    """
    if radius < 0:
        raise ValueError("Radius can't be negetive")
    return math.pi * (radius ** 2)
```
