# Advanced Topics

כשמתקדמים בפייתון, יש דברים יותר עמוקים שחשוב להבין. כאן תמצאו נושאים מתקדמים שיעזרו לכם לכתוב קוד יותר טוב ויותר מקצועי.

### מעט רקע

#### פייתון כשפה עילית (High-Level)

פייתון היא עילית, זאת אומרת שהיא קריאה ופשוטה. אבל תכלס מוסתר מאיתנו כול הניהול הזכרון המורכב ודרך המדוייקת שהמחשב עושה את הדברים. 
אנחנו נהנים מהפשטה של הדברים.

המשפט הכי חשוב שצריך להכיר כדי להבין את עומק השפה הוא:
> **"Everything in Python is an Object"**

זה אומר שכל דבר – מספרים, מחרוזות, פונקציות ואפילו מחלקות עצמן – הם אובייקטים. לכל אובייקט כזה יש "זהות", "סוג"

ניגע בזאת עוד בהמשך , אך חשוב שתכירו

#### Dunder Methods

כדי שהאובייקטים השונים בפייתון ידעו איך להתנהג ולתקשר אחד עם השני, פייתון משתמשת בפונקציות נסתרות  
הן מזוהות לפי התחביר שלהן: שתי שורות תחתונות לפני ואחרי השם (למשל `__init__`), ולכן הן נקראות גם **Dunder** (קיצור של Double Underscore).

השיטות האלו הן ה"חוזה" של פייתון. הן מגדירות מה קורה כשאנחנו מבצעים פעולות בסיסיות על אובייקט:

* **חיבור (+):** כשאתם כותבים `a + b`, פייתון למעשה קוראת לשיטה `a.__add__(b)`.
* **אורך (len):** כשאתם בודקים אורך של רשימה `len(my_list)`, פייתון קוראת ל-`my_list.__len__()`.
* **הדפסה (print):** כשמדפיסים אובייקט, פייתון מחפשת את `__str__` כדי לדעת איך להציג אותו כטקסט.



#### למה זה חשוב לנו?

זה מאפשר לנו להבין יותר לעומק אובייקטים מסויימים בשפה , ולשנות ולחקות אותם כדי לעשות דברים כפי שנרצה
